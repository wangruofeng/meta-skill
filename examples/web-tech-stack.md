# Web 技术栈第一性原理拆解

> 这是 `/first-principles` skill 的一段示例输出，用来展示它的产出长什么样。
>
> **输入**：「NewsNow 的技术栈 React + TanStack Router + TanStack Query + Jotai + UnoCSS + Vite + Nitro，我刚转 Web 不熟」

## 核心洞察

任何 Web 应用都只在回答 **6 个根本问题**，所有库都是这些问题的分工答案。看懂一个 Web 项目 = 看清它的库各自占了哪一格。

## 根本问题

| 根本问题 | 通俗说法 | NewsNow 的答案 | 常用度 |
| --- | --- | --- | --- |
| 界面怎么渲染 | 数据 → 可见页面 | React | ⭐⭐⭐⭐⭐ |
| 路由怎么走 | URL → 哪个页面 | TanStack Router | ⭐⭐⭐⭐⭐ |
| 数据怎么拿/存 | 前后端通信 | TanStack Query + ofetch | ⭐⭐⭐⭐⭐ |
| 状态怎么记 | 页面里的事怎么记 | Jotai | ⭐⭐⭐⭐ |
| 样式怎么写 | 怎么长得好看 | UnoCSS | ⭐⭐⭐⭐ |
| 怎么构建/部署 | 源码 → 可运行站点 | Vite + Nitro | ⭐⭐⭐⭐⭐ |

## 逐个拆解

### 问题 3：数据通信（分两层，新人最易混淆）

上层 **TanStack Query** 管「何时拿 / 怎么缓存」，下层 **ofetch** 管「怎么发请求」。一个管策略，一个管执行。

🚨 **关键易错点**：服务器数据用 Query，UI 状态（开关、输入框）用状态管理库（Jotai），别混。新人最容易把所有「数据」都塞进同一个地方。

### 问题 6：构建 + 部署

**Vite** 管「源码 → 前端产物」（开发体验、热更新、打包），**Nitro** 管「跑起来 + 部署到边缘」（SSR、serverless 适配）。一个管打包，一个管运行时。

## 心智模型与可迁移判断

**一句话心智模型**：库 = 分工。

**扫 `package.json` 七步法**（下次看任何 Web 项目都能复用）：
找 `react` → 路由库 → 数据层（query） → 状态库 → 样式库 → 构建工具 → 部署目标。七步走完，整个项目的骨架就清楚了。

**常用度分级**：
- 地基（必学）：React / 路由 / 数据请求 / 构建工具
- 进阶（按需）：状态管理（小项目可不学）/ 原子化 CSS
- 冷门（认得即可）：具体部署适配（Nitro）

## 边界与不确定

- 这个 6 格模型针对「典型」前后端分离 Web 应用；纯静态站、全栈一体框架（如 Next.js、Nuxt）会把几格合并到一起。
- 「常用度」基于通用 Web 开发，具体到你的业务领域可能不同（如重交互项目状态库常用度会上调）。
